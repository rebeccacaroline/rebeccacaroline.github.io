A Rehash of Arrays

An Array of Information

December 4, 2014

Rebecca Holzschuh

What is an array?

An array is an object that can hold a lot of data. Unlike a variable or a constant which only hold one piece of data at a time, an array can hold many pieces. For example, you can have an array of cats like so: 

cats = ["cheshire", "calico", "russian blue", "siamese", "persian", "sphynx"]

As you can see, instead of just one cat, we can have whole array of cats!

Data in an array is numbered. In order to access data in an array, you need to know the order of the data in the array. Data in arrays is numbered from 0 up. The 0th piece of data in this array is "cheshire." The way I would access this data would be: "cats[0]."

What is a hash?

A hash is similar to an array in that it can hold multiple pieces of data. However, a hash is different from an array in that the data is not numbered, and each piece of data is a key/value pair. This means that in a hash there are a certain amount of keys and each key corresponds to one value. Keys must be unique, but values do not have to be. Here is an example of a hash with cats: 

cats = {
          "favorite" => "russian blue",
          "ugly" => "sphinx",
          "scary" => "cheshire",
          "spoiled" => "siamese"
          }

With a hash, I don't need to know the order of the data or even the values, I just need to know the keys. I can ask Ruby to tell me which cat is scary by asking "puts cats["scary"]" and Ruby will tell me that the Cheshire cat is scary.

When to use an array over a hash

You want to use an array instead of a hash when you have an ordered list that you want to be able to keep track of by placement in a list. For example, if you know you will want to access the data in a list in an ordered manner, you'll want to use an array. This is because you can call cats[1] to get the second value in an array without knowing what the value is. Also if you know you'll want to iterate over values in a list in an ordered way, arrays are the way to go.

When to use a hash over an array

You want to use a hash instead of an array when you don't want your data to be in order. If you just want a collection of key/value pairs that are in no particular order (for example my cats hash) you would use a hash. In my cats hash, the ugly cat is not higher than the scary cat - each key/value pais is essentially at the same level.
