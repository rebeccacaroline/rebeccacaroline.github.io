Grouping Method

December 11, 2014

Rebecca Holzschuh

I chose to research the method Enumerable#group_by, because I had never seen or heard of it before. Basically, this method enables you to group values into a hash based on criteria that you can set. You can either give the method a code block and tell the method exactly how you want to group the numbers. For example, if you had all the numbers from 1-10 and you wanted to know how what the remainder of each number would be if you divided each number by 4, you could use #group_by and it would return a hash like so: 

puts (1..10).group_by {|i| i%4}
        {1=>[1, 5, 9], 2=>[2, 6, 10], 3=>[3, 7], 0=>[4, 8]}

As you can see, the output gives you a hash. The keys of the hash are the remainders, and the values are the numbers which, when divided by 4, give you that remainder. This is useful if you want to iterate over different groups within a collection of values. You don't have to give the method a code block though. If you don't give the method a code block, the method will return an Enumerator object, like so: 

puts (1..10).group_by
        #<Enumerator:0x007f8b3a9112f8>

You can also use #group_by with an array of strings, but you cannot use #group_by with just one string. One could use regular expressions in the code block, but that's another blog post for another day. Here's an example of an array of strings, that I'm grouping by which ones include the letter "a:" 

puts ["I", "am", "an", "amazing", "programmer"].group_by {|i| i.include?("a")}
        {false=>["I"], true=>["am", "an", "amazing", "programmer"]}
