Class v. Constructor

What is the difference?

January 15, 2015

Rebecca Holzschuh

JavaScript constructors are really confusing, so I'm going to try to explain them clearly here. Basically, constructors are like simple, less complex Ruby classes. Sort of.

Ruby classes are essentially groups of methods that create similar objects. For example, a Person class would create a new human being object, and it would have methods like #walk, #talk, and #think. Classes are clearly defined by the keyword "class." When you create a new object via a class, you would type "rebecca = Person.new." They're pretty hard to miss.

JavaScript constructors, on the other hand, are sneaky little devils. They look just like functions. The way you create one is like so: 

function Person () {
};

Looks like a function, right? The only way to tell if what you're looking at is a constructor is that the first letter is capitalized. But that's just the recommended style - you could potentially see a constructor without the first letter capitalized.

Classes and constructors do pretty similar things, in that they both create new objects that you can play around with. For example, here is an example of the above constructor, constructing a new object with my name: 

function Person (name) {
  this.name = name
};

var me = new Person("rebecca")
console.log(me.name)

//rebecca

The way JavaScript does this is with the prototype property. The prototype property is something that all JavaScript functions include, but most of the time nothing is really done with it. What this property does is enables the function to be used as a constructor. Therefore, any function can be a constructor.
